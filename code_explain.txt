# Flask Profit Predictor – Complete Code Documentation

## Overview

This document explains a Flask-based web application that predicts profit using a trained Machine Learning model. The application takes user input from a web form, processes it, sends it to the model, and displays the predicted result.

---

## Full Code

```python
from flask import Flask, render_template, request
import pickle
import numpy as np

app = Flask(__name__)

model = pickle.load(open("model2.pkl", "rb"))

@app.route("/")
def home():
    return render_template("index.html")

@app.route("/predict", methods=["POST"])
def predict():
    try:
        state_florida = int(request.form.get("State_Florida", 0))
        state_newyork = int(request.form.get("State_New York", 0))

        rd_spend = float(request.form["R&D Spend"])
        administration = float(request.form["Administration"])
        marketing = float(request.form["Marketing Spend"])

        final_input = np.array([[
            state_florida,
            state_newyork,
            rd_spend,
            administration,
            marketing
        ]])

        prediction = model.predict(final_input)[0]

        return render_template(
            "index.html",
            prediction_text=f"💰 Predicted Profit: {prediction:,.2f}"
        )

    except Exception as e:
        return render_template(
            "index.html",
            prediction_text=f"❌ Error: {str(e)}"
        )

if __name__ == "__main__":
    app.run(debug=True)
```

---

## 🔍 Detailed Explanation

### 1. Importing Libraries

* `Flask` → Used to create the web application.
* `render_template` → Used to load HTML pages.
* `request` → Used to get input data from the user.
* `pickle` → Loads the trained machine learning model.
* `numpy` → Used to create arrays for model input.

---

### 2. Creating Flask Application

* `app = Flask(__name__)`
* Initializes the Flask app.
* `__name__` helps Flask locate templates and static files.

---

### 3. Loading the Model

* `model = pickle.load(open("model2.pkl", "rb"))`
* Opens the saved model file in binary read mode.
* Loads the trained machine learning model into memory.

---

### 4. Home Route

* `@app.route("/")` "/" = homepage URL (like http://127.0.0.1:5000/)
                       When user opens this URL in browser → Flask calls home()
Whatever home() returns → shown in browser

"/" → this exact URL (homepage)
"/about" → http://127.0.0.1:5000/about
"/contact" → http://127.0.0.1:5000/contact
If you have only one file index.html, it will act as your homepage.

* Defines the main page of the application.
* When user visits the site, `index.html` is displayed.



---

### 5. Predict Route

* `@app.route("/predict", methods=["POST"])`
* Handles form submission.
* Accepts only POST requests.

Flow (very important 🔥)
1.User opens homepage:
http://127.0.0.1:5000/
2.index.html loads (your only page)
3. User fills form and clicks submit
4. Browser sends data to:
http://127.0.0.1:5000/predict

5. Flask runs:
def predict():
So /predict is:Not a separate HTML page

✅ A backend URL to process form data

---

### 6. Predict Function

* This function runs when the user submits the form.
* It processes input data and returns prediction.

---

### 7. Error Handling (try-except)

* Prevents application crash.
* If any error occurs, it shows the error message on the webpage.

---

### 8. Getting Input Data (Categorical)

* `request.form.get("State_Florida", 0)`
* Retrieves form data.
* Converts it into integer (0 or 1).
* Used for one-hot encoding.

---

### 9. Getting Input Data (Numerical)

* Converts user input into float values.
* Required because machine learning models work with numeric data.

Inputs:

* R&D Spend
* Administration
* Marketing Spend

---

### 10. Preparing Input Array

* `np.array([[...]])`
* Creates a 2D array.
* Machine learning models require input in this format.

---

### 11. Making Prediction

* `model.predict(final_input)`
* Sends input to model.
* Returns prediction as an array.
* `[0]` extracts the actual value.

---

### 12. Displaying Result

* Sends prediction back to HTML page.
* Uses formatted string:

  * `:,.2f` → shows number with commas and 2 decimal places.

Example:

* 123456.789 → 123,456.79

---

### 13. Handling Errors

* If any issue occurs:

  * Error message is displayed instead of crashing the app.

---

### 14. Running the Application

* `if __name__ == "__main__":`
* Ensures the app runs only when file is executed directly.
* `debug=True` enables:

  * Auto reload on changes
  * Detailed error messages

---

## Important Notes

### 1. Feature Order Must Match Model Training

The input order must be:
[Florida, NewYork, R&D Spend, Administration, Marketing Spend]

Incorrect order will give wrong predictions.

---

### 2. Avoid Spaces in Form Names

Example:
"State_New York" can cause errors.

Better:
"State_New_York"

---

### 3. Environment Consistency

* Train and run model in same environment.
* Different versions of libraries (like scikit-learn) may cause warnings or errors.

---

### 4. Virtual Environment Issue

If code works in conda but not in venv:

* Libraries may be missing
* Versions may differ
